# Lesson 01 - Extension Model and Ownership Boundaries

Status: Outline stub

## Why This Lesson Exists

Extension work fails when ownership boundaries are unclear. This lesson maps
what can be extended safely and what should remain stable core behavior.

## Planned Sections

- extension point taxonomy (service, packaging, installer surface)
- ownership boundaries across Sonic and sibling systems
- anti-patterns: shadowing core flows and undocumented overrides
- change proposal template for extension work

## Planned Hands-On

- classify candidate extension ideas by ownership boundary
- write one extension proposal with explicit non-goals
- identify contract checks needed before merge
